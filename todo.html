Developer Config

Git global config
Git is installed but unconfigured.
git-config.ps1 contents
git config --global user.name "Your Name" git config --global user.email "you@example.com" git config --global init.defaultBranch main git config --global pull.rebase true git config --global core.autocrlf input git config --global core.pager "delta" git config --global delta.navigate true git config --global delta.side-by-side true git config --global credential.helper manager
Also install delta: winget install dandavison.delta
high
SSH key generation
commands
ssh-keygen -t ed25519 -C "you@example.com" cat ~/.ssh/id_ed25519.pub | clip # copies to clipboard Start-Process "https://github.com/settings/keys"
Then set SSH for GitHub remotes:
git config --global url."git@github.com:".insteadOf "https://github.com/"

Package Management

delta — better git diffs
Pairs with the git config above. Syntax-highlighted, side-by-side diffs.
winget install dandavison.delta
medium
chezmoi — dotfiles management
Syncs $PROFILE, .gitconfig, Windows Terminal settings, VS Code settings across machines via a private git repo. Set up once, pays back forever.
commands
winget install twpayne.chezmoi chezmoi init --apply https://github.com/<you>/dotfiles
medium
pipx — isolated global Python tools
Prevents pip install --global conflicts. Better for CLI tools like Ansible, pre-commit, black.
commands
pip install pipx pipx ensurepath pipx install pre-commit pipx install ansible
low
Scoop — complement for PATH-friendly CLIs
No admin required, tools land on PATH immediately without reboot. Best for eza, zoxide, and any tool winget installs in awkward locations.
commands
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser irm get.scoop.sh | iex scoop install delta eza zoxide
low
pyenv-win — multiple Python versions
Useful if projects target different Python versions.
winget install pyenv-win.pyenv-win

System Tuning

Power plan — verify High Performance after reboot
laptop-setup.ps1 sets this. Also check Armoury Crate → set to Manual or Turbo on this ROG hardware.
powercfg /getactivescheme # should show High performance
medium
Taskbar cleanup
No reliable script for Windows 11. Right-click → Unpin for unused items. Priority: remove Widgets, Chat (Teams), Copilot (already done by debloat.ps1).
low
Default browser & file associations
Windows 11 requires per-extension changes — no single cmdlet. Settings → Apps → Default apps. Set Brave or Chrome as default, and VS Code for .json, .yaml, .tf, etc.
low
Windows Search — limit indexed locations
Default indexing crawls everything and spikes disk I/O. Settings → Privacy & security → Searching Windows → set to "Classic" or exclude node_modules, Program Files.
low
Scheduled WSL apt updates
The weekly task only covers Windows packages. Add a second task for WSL:
wsl -d Ubuntu -- sudo apt update -qq && sudo apt upgrade -y
low

WSL / Docker / Cloud

.wslconfig — tune memory for your workload
laptop-setup.ps1 creates a baseline: 8 GB / 4 CPUs / 2 GB swap. Adjust ~\.wslconfig based on actual usage. If you have 32 GB RAM, bump to 16 GB / 8 CPUs.
medium
Docker Desktop — enable WSL2 backend
After Docker Desktop installs, open it manually:
Settings → General → ✓ Use the WSL 2 based engine
Settings → Resources → WSL Integration → ✓ Ubuntu
high
kubeconfig — multi-cluster context management
When you add EKS/AKS/GKE clusters, merge contexts and use kctx (in powershell-profile.ps1) to switch.
merge command
$env:KUBECONFIG = "$HOME\.kube\config;$HOME\.kube\eks-prod" kubectl config view --merge --flatten > "$HOME\.kube\merged" Copy-Item "$HOME\.kube\merged" "$HOME\.kube\config"
info
